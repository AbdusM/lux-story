## Archive Maintenance

**When to add to archive:**
- Work session completed and superseded by newer docs
- Historical reference needed but not active development
- Duplicate documents consolidated

**When NOT to archive:**
- Active source of truth documents (keep in `docs/03_PROCESS/50-59/`)
- Frequently referenced implementation guides (keep in `docs/03_PROCESS/20-29/`)
- Core game design (keep in `docs/00_CORE/`)

**Archive naming convention:**
- Dated folders: `DDMMMYY_description/` or `MMMYYYY/`
- Session folders: `DDMMMYY_session/`
- Sprint folders: `DDMMMYY_sprint/`
- General month: `MMMYYYY/`
